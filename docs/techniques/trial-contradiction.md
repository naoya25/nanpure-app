# 仮定・矛盾（特定数字のシミュレーション）

ある **空マス**に、候補のひとつ **数字 `d` を仮に入れた**とみなし、**通常のルールだけ**（同じ行・列・ブロックから `d` を消すなど）を繰り返して盤を進めたとき、**矛盾**（同じユニットに同じ数字が二重、あるいはどこかの空マスの候補が空になる等）が出れば、「そのマスに `d` は入らない」と結論できる手筋。

**矛盾しなかったからといって、そのマスが `d` で確定するとは限らない**（別の仮定では矛盾する、など）。

## 初学者向け

### いつ使うか

メモを見ても **すぐ確定しない**が、「もしここが 6 だったら？」と試すと、あちこちで数字が決まっていき、**どこかでおかしい**（同じ行に 6 が 2 つ、など）ことがある。そのとき **最初に 6 と仮定したマスには 6 は入れない**と分かる。

### 注意

矛盾が出なかった場合、**たまたまその分岐が正解に近かった**だけかもしれません。だから **確定の根拠にはならない**（別の手筋が必要）。ヒント機能では「矛盾が出たときの候補削除」だけに使うと安全。

### 小さな例（流れ）

1. マス X の候補が `{6, 8}`。
2. X に `6` を仮置きして頭の中（またはコピー盤）で伝播。
3. ある行で `6` を置くべきマスが **なくなる**（すべてブロックされ矛盾）→ X は `6` 不可 → 候補は `{8}` へ。

## 開発者向け

### 概要アルゴリズム

候補付き盤（または確定盤＋候補）の **コピー**を取り、マス `c` に `d` を仮代入する。次を繰り返す:

1. 各ユニットで重複数字があれば **矛盾**。
2. 裸のシングル・隠れシングルなど **決定的なルールのみ**で伝播（実装方針による）。
3. 空マスで候補が空のマスがあれば **矛盾**。

矛盾なら「`c` から `d` を除外」。非矛盾なら **確定はしない**（オプションで深度制限付き探索に拡張できる）。

### 入力・出力

| 項目 | 内容                                       |
| ---- | ------------------------------------------ |
| 入力 | 候補グリッド、対象マス `c`、仮の数字 `d`   |
| 出力 | 矛盾時: `candidates[c]` から `d` を削除    |

### 実装上の注意

- コピーと再帰で **コストが高い**。ヒント 1 手に使うなら深度・適用回数を制限する。
- **Forcing chain** や **色分け**など、より強い論理は別モジュールに分けるとテストしやすい。
- UI では「仮定の説明」を短いメッセージキーに落とす。
