# 裸のシングル（Naked single）

**ある空マスを見て**、メモした候補が **1 つの数字だけ**になったとき、その数字でそのマスが確定する手筋。

文献の **隠れシングル**（ユニットと数字を決めて「その数字が置けるマスが 1 つ」）とは別物。独学で「隠れシングル」と呼んでいた場合、多くは **こちら（裸のシングル）** を指している。

[一覧に戻る](../sudoku-techniques.md)

---

## 初学者向け

### いつ使うか

そのマスにメモした数字が **ひとつだけ**のとき。「ここにはもう 8 しかない」と分かったら、そのマスは `8` で確定。

### 考え方

マス **ひとつ**に注目する。行・列・ブロックの制約で候補を削っていくと、**論理的にあり得る数字が 1 つに絞れる**ことがあります。ナンプレでは各マスは最終的に 1 つの数字なので、その時点で確定してよいです。

### 小さな例

ある空マスの候補メモが `{3, 7}` だったとします。別の手筋で `7` がこのマスから消え、メモが `{3}` だけになったとします。これが裸のシングルで、このマスは `3` 確定。確定したら、**同じ行・列・ブロック**の他のマスから `3` をメモから消す（**伝播**）。

## 開発者向け

### 定義

空マス `c` について `|candidates[c]| == 1` のとき、唯一の要素 `d` を `c` に確定する。

### 入力・出力

| 項目 | 出力                                                         |
| ---- | ------------------------------------------------------------ |
| 入力 | 候補グリッド                                                 |
| 出力 | 確定＋伝播（`c` の行・列・ブロックの他マスから `d` を削除） |

### アルゴリズム

```
for each cell c where empty(c):
  if |candidates[c]| == 1:
    d = the only candidate
    assign c = d
    remove d from candidates of all peers of c
```

### 実装上の注意

- 伝播の結果、**連鎖的に**別のマスが裸のシングルになるので、変化がなくなるまで繰り返すのが定石。
- バグりやすいのは「確定したのに候補から消し忘れ」なので、確定と削除は常にペアで処理する。
