# ラストセル（Full house / Last remaining cell）

ある **行・列・ブロック**（ユニット）の中に **空マスが 1 マスだけ**残っているとき、そのマスに入る数字は **まだ出ていない 1〜9 のうち残った 1 つ**に決まる、という手筋。

[一覧に戻る](../sudoku-techniques.md)

---

## 初学者向け

### いつ使うか

「このブロック（または行・列）を見たら、**空きが 1 マスだけ**」のとき。あと **どの数字がまだ無いか**を数えれば、空き 1 マスにその数字を入れられる。

### 考え方

ユニットには `1` から `9` まで **ちょうど 1 回ずつ**数字が入ります。8 マスが埋まっていれば、残り 1 マスには **出てきていない数字**が入るしかありません。

### 小さな例

ある行に `1,2,3,4,5,6,7,8` がすでにあり、空マスが 1 つだけなら、そこは必ず `9` です。ブロックや列でも同じです。

## 開発者向け

### 定義

ユニット `U` 内の空マスがちょうど 1 マス `c` のとき、`missing = {1..9} minus { values in U }` は要素 1 つ `d` の集合。`c` に `d` を確定する。

### 入力・出力

| 項目 | 内容                               |
| ---- | ---------------------------------- |
| 入力 | 確定盤（候補が無くても検出可能）   |
| 出力 | 1 マス確定＋伝播                   |

### アルゴリズム

```
for each unit U:
  emptyCells = { c in U | empty(c) }
  if |emptyCells| == 1:
    c = only cell in emptyCells
    d = only digit not present in U
    assign c = d, propagate
```

### 実装上の注意

- 確定盤だけで判定できるため、**候補初期化の前**にも走査できる。
- 隠れシングル・裸のシングルと結果は重なる局面もあるが、**カウントだけ**で済むので実装コストが低い。
