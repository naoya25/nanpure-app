# 裸のペア・裸のトリプル（Naked pair / triple）

同じ **行・列・ブロック**（ユニット）の中で、**2 マス**（または **3 マス**）だけに注目し、それらの候補が **同じ 2 個の数字だけ**（または **3 個の数字の組**だけ）に収まるときの手筋。その数字は **そのマスにしか入らない**ので、**ユニット内の他のマス**からその数字を候補から消せる。

[一覧に戻る](../sudoku-techniques.md)

---

## 初学者向け

### 裸のペア

例: ある行に空マスが複数あり、そのうち **2 マス**だけが、候補が `{3, 7}` と `{3, 7}`（同じ 2 数字だけ）だとします。この行では `3` と `7` は **この 2 マスで使い切る**ので、**同じ行の他のマス**から `3` と `7` をメモから消してよいです。

### 裸のトリプル

**3 マス**の候補の和集合が **ちょうど 3 種類の数字**に収まり、かつ論理的に「この 3 マスでこの 3 数字を使い切る」構造（厳密な定義は実装時に固定）のとき、**ユニット内の他マス**からその 3 数字を消します。見つけにくいので上級寄り。

### 小さな例（ペア）

ある行の一部（`.` は他に空マスあり）:

```
[ {3,7} ] [ {3,7} ] [ {2,3,7,9} ] ...
         ↑ このマスから 3,7 を消せる（左の 2 マスが 3 と 7 を占有）
```

## 開発者向け

### 裸のペア

ユニット `U` 内の異なる空マス `a, b` で、`candidates[a] == candidates[b]` かつ `|candidates[a]| == 2` のとき、数字集合 `P = candidates[a]` について、`U` 内の `a,b` 以外のマス `c` から `P` の各数字を候補削除。

### 裸のトリプル

ユニット内の 3 マスの組 `(c1,c2,c3)` で、候補の和集合が 3 数字 `T` に収まり、かつ subset 制約（各マスは `T` の部分集合、かつ列挙可能な「3 マス 3 数字」のパターン）を満たすとき、`U` の他マスから `T` を削除。実装では **有効な 3 マス組を列挙**し、条件を満たすものだけ適用する。

### 入力・出力

| 項目 | 内容                 |
| ---- | -------------------- |
| 入力 | 候補グリッド         |
| 出力 | 候補削除（確定は不定） |

### 実装上の注意

- トリプルは **定義の取り違え**でバグが出やすい。列挙パターンをコメントとテストで固定する。
- ペアは O(空マス²) でユニットあたり十分高速。
