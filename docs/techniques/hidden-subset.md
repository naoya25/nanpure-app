# 隠れペア・隠れトリプル（Hidden pair / triple）

ユニット内で、ある **2 つの数字**（または **3 つ**）について、「それぞれが候補として現れうるマス」が **共通してちょうど 2 マス**（または **3 マス**）に限られるとき、そのマスには **他の数字は入らない**ので、候補をその数字（たち）だけに **絞り込める**手筋。

独学で「隠れペア」と呼ぶ場合、2 マス版に加え **3 マス・3 数字**もまとめて覚えていることが多い（本項では **隠れトリプル**として扱う）。

## 初学者向け

### 隠れペアのイメージ

ある行で、数字 `4` が候補に残っているのはマス A と B だけ、数字 `9` も候補に残っているのは A と B だけ、だとします。するとこの行の `4` と `9` は **A と B で使い切る**ので、A と B のメモから **`4,9` 以外を全部消してよい**です（まだ `{4,9}` 以外がメモにあれば矛盾ではなく単なる削減）。

### 隠れトリプル

3 つの数字が、候補として現れるマスの組合せが **ちょうど 3 マス**に収まるとき同様に、その 3 マスの候補をその 3 数字に絞る。発見が難しく上級。

### 小さな例（ペア）

- マス A の候補: `{2,4,9}`
- マス B の候補: `{4,5,9}`

この行で `4` と `9` が候補に出るのが A, B **だけ**だと分かったら、A, B は最終的に `4` と `9` を使うマスなので、A は `{4,9}`、B は `{4,9}` に削れる（`2` と `5` を削除）。

## 開発者向け

### 隠れペア

ユニット `U`、異なる数字 `d1, d2` について:

- `S1 = { c in U | empty(c) and d1 in candidates[c] }`
- `S2 = { c in U | empty(c) and d2 in candidates[c] }`

`S1 == S2` かつ `|S1| == 2` のとき、`S1` の各マス `c` について `candidates[c]` を `candidates[c] ∩ {d1, d2}` に置き換える（2 数字以外を削除）。

### 隠れトリプル

3 数字 `d1,d2,d3` について、各 `di` の候補マス集合の「和」がちょうど 3 マスに一致し、かつ各そのマスが 3 数字のいずれかを含む、など **厳密条件**を満たす組を列挙し、該当 3 マスの候補を `{d1,d2,d3}` に制限。

### 入力・出力

| 項目 | 内容                               |
| ---- | ---------------------------------- |
| 入力 | 候補グリッド                       |
| 出力 | 候補の **削減**（必ずしも確定しない） |

### 実装上の注意

- 隠れペアは数字ペア × ユニットで総当たり可能。
- トリプルは組合せ爆発に注意し、早期フィルタ（候補マス数）を入れる。
