# 候補の全メモ（ペンシルマーク）

空マスに「入りうる数字」を **すべて小さく書く**習慣。単一の論理手筋というより **中級に入るための土台**。以降の裸のシングル・隠れシングル・ペアなどは、多くの場合 **候補が揃った盤面**で見つけやすい。

[一覧に戻る](../sudoku-techniques.md)

---

## 初学者向け

### 何をするか

各 **空マス**について、**同じ行・同じ列・同じ 3×3 ブロック**にすでに出ている数字は入れられないので、残った `1〜9` のうち **まだ禁止されていない数字**をすべてメモする。

### やり方のコツ

1. まず **確定している数字**だけを頼りに、各空マスに候補を書く（交差のブロックも忘れずに）。
2. **隠れシングル**や **ポインティング**で候補が削れたら、メモを消し足しする。
3. メモが **1 つだけ**になったマスがあれば、それが **裸のシングル**（その数字で確定）。

### 小さな例

ある空マスを見たとき、行には `1,3`、列には `2,5`、ブロックには `1,4` があるとします。使えない数字は `{1,2,3,4,5}` なので、候補は `{6,7,8,9}`。この 4 つをメモします。さらに論理で削っていき、例えば `{7}` だけになればそのマスは `7` 確定です。

## 開発者向け

### 候補の初期化（アルゴリズム）

確定盤 `grid[81]`（空は `0`）が与えられたとき、各空マス `i` について:

```
candidates[i] = {1..9}
  minus { grid[j] for j in peers(i) where grid[j] != 0 }
```

`peers(i)` は同じ行・同じ列・同じブロックの他 20 マス（実装では 27 ユニットのメンバから導出）。

### データ構造

- UI と共有するなら **9 ビット bitmask**（ビット `d` が数字 `d`）が扱いやすい。
- 人間向け表示はビットを `1..9` に展開。

### メモとの対応

「全メモ」は **初期候補＋各テクニックによる削除**の結果として機械的に保てる。ユーザーが手でメモを消す操作は、内部では同じ `candidates` の更新としてモデル化できる。

### 関連テクニック
