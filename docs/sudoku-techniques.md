# ナンプレ解法テクニック（一覧）

論理だけで盤面を進めるための手筋を整理する。

## 難易度の目安


| 難易度  | 目安                                       |
| ---- | ---------------------------------------- |
| 初級   | メモなしでも自明な手筋。                             |
| 中級   | 候補の整理が前提になる手筋。ほとんどのナンプレがここまでのテクニックのみで解ける |
| 上級   | ほとんどのナンプレを解くのに必要ない手筋。                    |
| 難問向け | 手順が長く説明コストが高い。機械的な全探索に近い。人間向きではない。       |
| 探索   | 仮置き・矛盾。                                  |


## テクニック一覧


| 難易度  | テクニック                                | 概要                                                                                               | 詳細  | 実装状況 |
| ---- | ------------------------------------ | ------------------------------------------------------------------------------------------------ | --- | ---- |
| 初級   | フルハウス                                | 行・列・ブロックのどれかで空きマスが 1 つだけ → 未出現の数字を確定。隠れシングルに包含される                                                | -   | done |
| 初級   | シングル                                 | ピアの確定数字からそのマスの候補を求め、候補が 1 つだけなら確定（メモは候補との交差）。                                                    | -   | done |
| 初級   | 隠れシングル                               | あるユニット内で、ある数字が置けるマスが 1 つだけ → 確定。                                                                 | -   | done |
| 初級   | メモ1候補の確定                             | ピアは見ず、メモ（ペンシルマーク）がちょうど 1 桁だけの空マスをその数字で確定。                                                        | -   | done |
| -    | 正解同期                                 | テクニックではない。プレイ支援用の 1 手。確定マスが正解と異なればその数字だけ消す（メモは触れない）。                                             | -   | done |
| 中級   | ペンシルマーク（候補の記入）                       | テクニックというより作業法。多くの中級以上の手筋の前提。                                                                     | -   | done |
| 中級   | ポインティング                              | ブロック内で某数字の候補が 1 行（または 1 列）にだけある → その行（列）のブロック外から当該数字の候補を削除。                                      | -   | done |
| 中級   | ボックス・ライン削減                           | 某行（列）で某数字の候補が 1 ブロック内にだけある → そのブロックの当該行（列）以外から削除。                                                | -   | done |
| 中級   | ペア                                   | ユニット内の 2 マスで候補の和集合がちょうど 2 種類（各マスはその 2 種類のみ）→ その数字をユニット内の他マスから削除。                                 | -   | done |
| 中級   | トリプル                                 | 同上（3 マス・3 種類）。                                                                                   | -   | done |
| 中級   | クァッド                                 | 同上（4 マス・4 種類）。                                                                                   | -   | done |
| 中級   | 隠れペア                                 | 某 2 種類の数字の候補がユニット内でちょうど 2 マスにだけ現れる → それらのマスから他数字の候補を削除。                                          | -   | done |
| 中級   | 隠れトリプル                               | 同上（3 種類・3 マス）。                                                                                   | -   | done |
| 中級   | 隠れクァッド                               | 同上（4 種類・4 マス）。                                                                                   | -   | done |
| 中級   | fish22                               | 某数字について 2 行×2 列（または転置）に候補が拘束する基本魚 → それ以外から削除。別名: **X-Wing**。                                     | -   | done |
| 中級   | fish33                               | 同上の 3×3。**Swordfish**。                                                                           | -   | done |
| 中級   | スカイスクレーパー                            | 某数字について、同一の行（または列）上の 2 本の強いリンクが、別の列（行）で「足」側のマスを共有する形 → 屋根から見えるマスで削除（フロア上の共役マスは除外）。               | -   | done |
| 中級   | ツーストリング・カイト（`TWO_STRING_KITE`）       | 某数字について、ある行とある列にそれぞれ共役ペアがあり、交わるブロックのマスで繋がる凧形。凧の両端を同時に見るマスから当該数字を削除。                              | -   | done |
| 上級   | fish44                               | 同上の 4×4。**Jellyfish**。                                                                           | -   | done |
| 上級   | fish55                               | 同上の 5×5（一般名は文献により異なる）。超大魚は超レア。                                                                   | -   | done |
| 上級   | fish66                               | 同上の 6×6。                                                                                         | -   | done |
| 上級   | fish77                               | 同上の 7×7。                                                                                         | -   | done |
| 上級   | fish88                               | 同上の 8×8（盤サイズ上、基本魚としてはここまで）。                                                                      | -   | done |
| 上級   | フィン付き・殺進魚・変形魚など                      | 基本魚に例外マスやボックス拘束を加えた拡張。文献・実装で名称が細分化される。                                                           | -   | yet  |
| 上級   | ターボフィッシュ                             | 単一数字の **強リンク-弱リンク-強リンク**（S-W-S）連鎖。row/col/block の強リンクを使い、Skyscraper / Two-String Kite を含む一般形で削除。 | -   | done |
| 上級   | XY-Wing（`XY_WING`）                   | ピボット `{x,y}` と翼 `{x,z}` / `{y,z}` の構造。2 つの翼を同時に見るマスから `z` を削除。                                   | -   | done |
| 上級   | XYZ-Wing（`XYZ_WING`）                 | ピボット `{x,y,z}` と翼 `{x,z}` / `{y,z}`。ピボットと 2 翼を同時に見るマスから `z` を削除。                                 | -   | done |
| 上級   | WXYZ-Wing（`WXYZ_WING`）               | 単一ユニット内 4 マスで候補和集合が 4 種の ALS 形（restricted 候補あり）として扱い、他候補を同時視認マスから削除。                             | -   | done |
| 上級   | W-Wing（`W_WING`）                    | 同じ二値候補 `{x,y}` の 2 マスと、候補 `x` の強リンクで両端を繋ぎ、2 マスを同時に見るセルから `y` を削除。                 | -   | done |
| 上級   | ユニーク長方形（`UNIQUE_RECTANGLE`）       | 一意解を壊す候補配置を避けて削除・確定。現実装は **Type 1**（4隅のうち3セルが同一二値、残り1セルから二値を削除）。                 | -   | done |
| 上級   | BUG+1（`BUG_PLUS_1`）                 | 空マスがほぼ二値で 1 マスだけ三値のとき、行・列・ブロックで 3 回現れる候補を残し、当該マスの他候補を削除（一意解を仮定）。            | -   | done |
| 難問向け | シングル・カラー／マルチカラー／3D Medusa など         | 強いリンクを色分けし、同色同士の矛盾や「どちらかの色が真」から削除。                                                               | -   | yet  |
| 難問向け | XY-Chain（`XY_CHAIN`）                | 二値セルを弱リンク（同候補共有）とセル内強リンクで交互に連鎖し、同じ候補で閉じる両端を同時に見るマスからその候補を削除。                                  | -   | done |
| 難問向け | X-Chain（`X_CHAIN`）                  | 単一数字候補で、強リンク/弱リンクを交互に連鎖し、強リンク始端と終端を同時に見るマスから当該数字を削除。                                    | -   | done |
| 難問向け | X-Cycle（`X_CYCLE`）                  | 単一数字候補の強/弱リンクで閉路（nice loop）を作り、continuous では弱リンク辺の共有ハウスから、discontinuous（弱-弱）では該当ノードから候補を削除。      | -   | done |
| 難問向け | AIC                                  | 強い・弱いリンクの交互列を一般化して扱う。X-Chain より広い枠組み。                                                             | -   | yet  |
| 難問向け | ALS-XZ / ALS-XY-Wing / Sue de Coq など | ほぼロック集合の共通候補からの削除。                                                                               | -   | yet  |
| 探索   | 仮置き・矛盾（試し打ち）                         | 候補を仮に真とし伝播、矛盾なら反対が真。**探索**に近く強力。                                                                 | -   | yet  |
