# ナンプレ解法テクニック（機械実装用メモ）

プレイ画面への **ヒント・自動メモ削り・解法支援** などを、後から **純粋関数として積み上げる** ための整理。ビジネスルールや UI 文言は別途。本書は **ロジックの単位・入出力のイメージ** を揃える。

既存コードとの対応: 盤は 81 マス・インデックス `0..80`（`lib/validates/grid.ts` の `SUDOKU_CELLS`）、行 `floor(i/9)`、列 `i % 9`、ブロック（3×3）は `floor(row/3)*3 + floor(col/3)` で番号 `0..8`。

---

## 1. 内部表現（実装の前提）

### 1.1 確定値と候補

| 概念 | 例 | 備考 |
|------|-----|------|
| **確定値** | `0` = 空、`1..9` = 埋まり | 現状の `grid` と同じ |
| **候補** | 各空マスに「まだあり得る数字」の集合 | ビットマスク 9bit（メモ UI と同型にできる）または `Set<1..9>` |

機械処理の基本ループ例:

1. ヒント・ユーザー入力から **確定盤** を作る。
2. 各空マスについて、**同じ行・列・ブロックに既に出ている数字を除いた集合** を候補の初期値とする。
3. テクニックは「候補の削除」または「マスへの確定（＝そのマスの候補をその数字のみに）」を返す。

### 1.2 ユニット（unit）

行・列・ブロックのいずれか。**各ユニットはちょうど 9 マス**で、数字 `1..9` がそれぞれ 1 回ずつ現れる制約の単位。

実装では「マスインデックスの配列（長さ 9）」のリストを 27 本（行 9 + 列 9 + 塊 9）持てばよい。

### 1.3 テクニック関数のイメージ（純粋）

後述の各テクニックは、概ね次のいずれかの形で表せる。

- **候補の更新**: `(candidates: CandidateGrid, grid: Grid) → { candidates, changed }`
- **一手の提示**（UI 用）: 種類・対象マス・根拠マス・メッセージキー などの構造体の配列

`CandidateGrid` は `81` 要素、各要素は「空マスなら候補、確定マスなら無視 or 空集合」。

## 2. 単一候補系（最優先・人間でも機械でも取りやすい）

### 2.1 ラストセル（Last remaining cell / Full house）

**ユニット内に空マスが 1 つだけ**のとき、まだ出ていない数字 `1..9` のうちそのユニットに未出現のものはちょうど 1 つなので、その空マスに確定する。

| 項目 | 内容 |
|------|------|
| 入力 | 確定盤（または候補） |
| 出力 | 1 マスへの確定 |
| 備考 | 「そのユニットの空きマス数」と「未使用数字」を数えるだけ |

### 2.2 裸のシングル（Naked single）

ある空マスの **候補が 1 個だけ**のとき、その数字で確定する。

| 項目 | 内容 |
|------|------|
| 入力 | 候補グリッド |
| 出力 | 1 マスへの確定 → 同じ行・列・ブロックからその数字を候補から削除（伝播） |
| 備考 | 候補更新の直後に毎回走査してよい |

### 2.3 隠れシングル（Hidden single）

あるユニットの中で、数字 `d` が **候補として現れるマスが 1 つだけ**のとき、そのマスは `d` で確定する。

| 項目 | 内容 |
|------|------|
| 入力 | 候補グリッド |
| 出力 | 1 マスへの確定 + 伝播 |
| 備考 | ユニット × 数字 `1..9` の二重ループで機械的に検出可能 |

## 3. subset 系（ペア・トリプル）

### 3.1 裸のペア（Naked pair）

同一ユニット内の **異なる 2 マス**が、候補集合が同じ **ちょうど 2 数字**（例 `{3,7}`）だけを持つとき、その 2 数字はこの 2 マスにしか入らないので、**ユニット内の他マス**からその 2 数字を候補から消す。

### 3.2 裸のトリプル（Naked triple）

同一ユニット内の **3 マス**について、候補の和集合が **3 種類の数字に収まり**、かつ subset の条件（各マスがその 3 数字のみ、かつ「3 マスで 3 数字を使い切る」構造）を満たすとき、ユニット内の他マスからその 3 数字を消す。

※定義の厳密化は実装時に「列挙パターン」をコードコメントに固定するとバグが減る。

### 3.3 隠れペア / 隠れトリプル（Hidden pair / triple）

ユニット内で、ある 2（または 3）個の数字が **それぞれ現れる候補マスの集合**の共通部分が、ちょうど 2（または 3）マスに限定されるとき、そのマスの候補をその数字群だけに絞る（他候補を削除）。

---

## 4. 交差法（ブロックと行・列）

### 4.1 ポインティング（Pointing / ブロック→行・列）

ある **ブロック**内で、数字 `d` の候補が **同じ行（または同じ列）上のマスにだけ**存在するとき、その行（列）の **ブロック外**のマスから `d` を消す。

### 4.2 ボックス・ライン削減（Claiming / 行・列→ブロック）

ある **行**（または **列**）で、数字 `d` の候補が **1 つのブロック内**にだけ存在するとき、そのブロックの **当該行（列）以外**のマスから `d` を消す。

---

## 5. 魚系（中級〜）

### 5.1 X-Wing

ある数字 `d` について、**2 行**において `d` の候補位置が **同じ 2 列にだけ**あるとき（または行と列を入れ替えた対称形）、その 2 列の **他行**から `d` を消す。

### 5.2 ソードフィッシュ（Swordfish）

X-Wing の 3 行 × 3 列版（一般化）。

---

## 6. 実装の進め方（提案）

1. **候補の初期化**（確定盤から行・列・ブロックで除外）を 1 関数にまとめる。
2. **伝播**（確定したマスの数字を関連ユニットから削除）を共通処理にする。
3. テクニックは **難易度の低い順**に追加し、それぞれ **単体テスト**（小さい盤面・既知の手筋）を付ける。
4. UI では「ユーザー盤」と「非表示の solution」は別用途。**ヒントは候補と論理だけ**にし、正解文字列は「検証用」に閉じるとチートになりにくい（方針はプロダクトで決定）。
