## 独学用語と標準用語（すばやい対照）

| 本人の言い方                            | 標準的な呼び方                                         |
| --------------------------------------- | ------------------------------------------------------ |
| ブロック内 / 列・行内のシングル         | **隠れシングル**（ユニットがブロック・行・列）         |
| 2 段階のシングル                        | **ポインティング**（＋対になる交差＝ボックス・ライン） |
| 中級でいう「隠れシングル」（候補 1 つ） | **裸のシングル**                                       |
| ペア・トリプル                          | **裸のペア / 裸のトリプル**                            |
| 隠れペア（2 または 3 マス版）           | **隠れペア / 隠れトリプル**                            |
| 特定数字のシミュレーション              | **仮定・矛盾**（単一数字の試し打ち）                   |

## 難易度ラベル（ヒント文言・補助）

DB の `level` 帯や UI の「初級/中級/上級」と並べるときの **目安**。

| 段階     | 意味                                                                                     |
| -------- | ---------------------------------------------------------------------------------------- |
| **初級** | **メモなし**でも、ブロック・行・列の眺めと単純なカウントで進みやすい。一覧の初級が中心。 |
| **中級** | **全メモ**や候補が揃った状態だと、裸のシングルや隠れシングルが取りやすい。               |
| **上級** | **メモがあっても** subset や仮定・矛盾など、パターン認識や探索が要る。一覧の上級が中心。 |

**補足**: 文献の **隠れシングル** = ユニットと数字 `d` を決め、「`d` が候補に残るマスが 1 つだけ」。**裸のシングル** = 「マスを見て候補が 1 つ」。どちらも実装では候補グリッドから検出する。
