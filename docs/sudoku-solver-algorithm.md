# 求解アルゴリズム（概要）

ナンプレの答えを求めるアルゴリズムです。

アプリ側のプログラムがこの考え方に沿って計算します。**問題生成**（[ナンプレ問題の生成方法](./create-sudoku.md) のステップ 2・3）や **一意解の確認**でも同じ枠組みを使います。

## 考え方（ざっくり）

1. **整合チェック**
   行・列・3×3 ブロックごとに、1〜9 が重複していないかを見ます（空マス `0` は複数あってよい）。

2. **次に埋めるマスの選び方**
   空いているマスのうち、そのマスに属する行・列・ブロックに **すでに出ている数字の種類が最も多いマス**を選びます。そこに入れられる候補（まだ現れていない 1〜9）だけを試します。探索が進みやすいマスから手を付ける、というヒューリスティックです。

3. **深さ優先探索**
   候補ごとに盤を更新し、**スタック**で状態を積みながら完成形が得られるまで進めます。求める解の個数に上限（例: 1 件だけ欲しい、最大 2 件まで見て一意か判定する）を付け、それに達したら打ち切ります。

人が手で解くときの「テクニック名付きの解法」とは別物で、**機械的な探索**として動かしています。
